What is a Quantum Network?

A quantum network connects quantum nodes (stationary qubits) through quantum channels to distribute entanglement and quantum information. Unlike classical networks, quantum networks can leverage unique quantum properties like:

  • Quantum entanglement for secure communication
  • Quantum teleportation for state transfer
  • Distributed quantum computing capabilities

The QuREBB Framework

Our simulator is built on the QuREBB (Quantum Remote-Entanglement Building Blocks) framework, which provides a modular approach to quantum network simulation:

Layer 1: Logical Building Blocks & Topology

The highest level of abstraction that defines:

  • Network topology (sender-receiver, detection-in-midpoint, source-in-midpoint)
  • Logical operations (photon sources, spin-photon interfaces, detection)

Layer 2: Encoding

Defines how quantum information is encoded:

  • Photonic encodings (Fock-state, time-bin, polarization, etc.)
  • Spin qubit encoding in different physical systems

Layer 3: Physical Building Blocks

Concrete implementation mechanisms:

  • Spontaneous emission, coherent scattering
  • Reflection-based interactions
  • Quantum gates and measurements

Layer 4: Quantum Optical Modeling

Most detailed level, including:

  • Cavity QED parameters and noise models
  • Numerical simulations of quantum dynamics
  • Detailed physical parameters

Entanglement Protocols

Remote-Entanglement Protocols (REPs) are methods to generate quantum entanglement between distant qubits. Our simulator focuses on three main protocols:

Protocol A: Detection-in-Midpoint

Both endpoints generate spin-photon entanglement, and photons are measured at a midpoint detector using Bell State Measurement (BSM).

  • Advantages: Simpler implementation, reduced classical communication
  • Uses Fock-state encoding (single photon/vacuum)
  • Typically implemented with NV centers, quantum dots, trapped ions

Protocol B: Sender-Receiver

First node creates spin-photon entanglement and sends the photon to the second node where it interacts with the spin through conditional reflection.

  • No intermediate station required
  • Uses time-bin encoding for robustness
  • Implemented with atoms in optical cavities or color centers

More Protocols & Implementations

Protocol C: Time-Bin with Detection-in-Midpoint

Similar to Protocol A but using time-bin encoding for improved robustness to channel noise.

  • Better resilience to optical phase fluctuations
  • Compatible with weak coherent pulses as photon source
  • Requires precise timing synchronization

Physical Implementations

Our simulator can model different physical systems for quantum network nodes:

Diamond Color Centers

  • NV centers
  • SiV centers
  • Long spin coherence times
  • Room temperature operation

Trapped Atoms/Ions

  • Neutral atoms in cavities
  • Trapped ions (e.g., Yb+)
  • High-fidelity operations
  • Excellent coherence properties

Semiconductor QDs

  • Strong optical transitions
  • Integration with photonics
  • Potential scalability
  • Challenges with indistinguishability

Node & Edge Implementation

NodeItem Class

Represents a quantum node (stationary qubit) in the network:

  • Customizable type: memory, detector, memory-detector, repeater
  • Quantum properties: Number of qubits, coherence time
  • Physical properties: Qubit technology, insertion loss
  • Visual representation: Different shapes based on node type
  • Interactive: Movable, selectable, context menu for properties
class NodeItem(QGraphicsPolygonItem):
    def get_properties(self):
        return {
            "name": self.node_name,
            "type": self.node_type,
            "num_qubits": self.num_qubits,
            "qubit_tech": self.qubit_tech,
            "coherence_time": self.coherence_time,
            "insertion_loss": self.insertion_loss,
        }

EdgeItem Class

Represents a quantum channel between nodes:

  • Automatic distance calculation: Computes distance between nodes
  • Physical properties: Fiber loss rate, transmission efficiency
  • Visual feedback: Updates dynamically when nodes move
  • Distance label: Shows physical distance in kilometers
class EdgeItem(QGraphicsLineItem):
    def calculate_distance(self):
        """
        Calculate distance between nodes in km
        """
        p1 = self.source_node.scenePos()
        p2 = self.target_node.scenePos()
        dx = p2.x() - p1.x()
        dy = p2.y() - p1.y()
        distance_pixels = math.hypot(dx, dy)
        scale_factor = 10.0  # 10 pixels = 1 km
        return distance_pixels / scale_factor

Simulation Fundamentals

Our simulator bridges the gap between the GUI representation and the physical quantum processes:

Network State Extraction

To run a simulation, we extract the network state from the GUI:

def extract_network_state(self):
    """
    Extract the state of the network as 
    a dictionary of nodes and links.
    """
    nodes = {}
    links = []
    for item in self.items():
        if isinstance(item, NodeItem):
            nodes[item.label_text] = item.get_properties()
        elif isinstance(item, EdgeItem):
            links.append(item.get_properties())
    return {"nodes": nodes, "links": links}

This structured representation contains all quantum properties needed for simulation.

Protocol A - Mathematical Details

Protocol A (Detection-in-Midpoint) uses Fock-state encoding and spontaneous emission to generate entanglement:

Quantum State Evolution

1. Both nodes create spin-photon entanglement through spontaneous emission:

|Ψ⟩A = 1/√2(|0⟩A|0⟩pA + |1⟩A|1⟩pA)

|Ψ⟩B = 1/√2(|0⟩B|0⟩pB + |1⟩B|1⟩pB)

2. Combined system state before Bell-state measurement:

|Ψ⟩combined = 1/2(|0⟩A|0⟩pA + |1⟩A|1⟩pA)(|0⟩B|0⟩pB + |1⟩B|1⟩pB)

3. After Bell-state measurement at the midpoint, the spin state becomes (for Φ+ outcome):

|Ψ⟩final = 1/√2(|0⟩A|0⟩B + |1⟩A|1⟩B)

Key Performance Metrics

Fidelity Calculation:

F = F0 · e-t/T2

where:

  • F0 is the initial fidelity
  • t is the photon travel time
  • T2 is the coherence time

Rate Calculation:

R = Rattempt · Psuccess

Psuccess = ηA · ηB · ηBSM

where:

  • ηA, ηB are the efficiencies for nodes A and B
  • ηBSM is the Bell-state measurement efficiency

In our simulator implementation, these mathematical principles are encoded in the quantum channels and density matrix operations of the QuREBB framework.

Cavity QED Model

For accurate quantum simulation, we model the light-matter interaction in cavity quantum electrodynamics (cavity QED):

Key Parameters

g: Coupling strength (vacuum Rabi frequency)

κ: Cavity decay rate

γ: Emitter decay rate

C = 4g²/(κγ): Cooperativity

The cooperativity C determines how strongly the emitter couples to the cavity mode. Higher cooperativity leads to better performance in protocols B and C.

Realistic Values (SiV in Diamond)

  • g ≈ 8.38 GHz (emitter-cavity coupling)
  • κ ≈ 21.8 GHz (cavity decay rate)
  • γ ≈ 92.5 MHz (emitter decay rate)
  • γ* ≈ 30.5 MHz (pure dephasing rate)
  • C ≈ 105 (cooperativity)

State-Dependent Reflection

For conditional reflection in Protocol B, the reflection coefficient depends on the spin state:

r|1⟩ = 1 - 2κr/(κ + i(ωc - ω) + g²/(γ/2 + i(ωa - ω)))

r|0⟩ = 1 - 2κr/(κ + i(ωc - ω))

where:

  • κr is the cavity-waveguide coupling rate
  • ωc is the cavity resonance frequency
  • ωa is the atomic transition frequency
  • ω is the frequency of the input photon

Purcell Enhancement

The cavity enhances spontaneous emission through the Purcell effect:

Fp = 4g²/κγr = (3/4π²)(Q/V)(λ/n)³

where Q is the cavity quality factor, V is the mode volume, λ is the wavelength, and n is the refractive index.

This enhancement improves both the rate and fidelity of our entanglement protocols.

Quantum Channel Representation

In our simulator, quantum operations are modeled as quantum channels (completely positive trace-preserving maps):

Spontaneous Emission PBB

ρout = ĈρinĈ†

Ĉ = √pcohĈcoh + √plossĈloss + √pincohĈincoh + √p2phĈ2ph

where:

  • Ĉcoh = |0s⟩⟨0s| ⊗ Î + |1s⟩⟨1s| ⊗ â† (coherent channel)
  • Ĉloss = |1s⟩⟨1s| ⊗ â†loss (loss channel)
  • Ĉincoh = |1s⟩⟨1s| ⊗ â†incoh (incoherent channel)
  • Ĉ2ph = |1s⟩⟨1s| ⊗ (1/√2)â†â† (double excitation)

Physical Parameters for Channel Probabilities

pcoh = ⟨1|ρin|1⟩ · (κr/κ) · [C/(C+1)] · [Fpγr/(Fpγr + γ*)] + ⟨0|ρin|0⟩

where:

  • κr/κ is the ratio of cavity output coupling
  • C/(C+1) is the emitter-cavity coupling efficiency
  • Fpγr/(Fpγr + γ*) is the coherent emission fraction

Photonic Loss Channel

ρout = trL[Û(ρin ⊗ |0L⟩⟨0L|)Û†]

Û = exp[θL(âa†L - â†aL)]

θL = arcsin(√L)

where L is the loss probability (0 to 1) calculated from fiber length and loss rate.

Simulation Results Dialog

Protocol Comparison

Our simulations show different trade-offs between protocols:

  • Protocol A: Highest rate for short distances
  • Protocol B: Better fidelity at longer distances
  • Protocol C: Most robust to phase noise

The optimal protocol depends on network topology, distance, and available hardware.

Sample Code: Result Processing

def handle_simulation_result(self, parameters):
    """Handle the result of a successful simulation."""
    try:
        current_scene = self.current_scene()
        if not current_scene:
            QMessageBox.warning(self, "Error", "No active canvas.")
            return

        network_state = current_scene.extract_network_state()
        results = simulate_protocol_a(
            network_state,
            coherence_time=parameters["T2"],
            transmission_efficiency=parameters["eta"],
            detection_efficiency=parameters["Pdetect"],
            fiber_loss=parameters["fiber_loss"],
            qubit_type=parameters["qubit_type"],
            bsm_setting=parameters["bsm_setting"],
        )
        
        if results:
            dialog = ResultsDialog(results, parameters, self)
            dialog.exec_()
        else:
            QMessageBox.critical(self, "Error", "No results returned.")
    except Exception as e:
        QMessageBox.critical(self, "Error", f"An error occurred: {str(e)}")
