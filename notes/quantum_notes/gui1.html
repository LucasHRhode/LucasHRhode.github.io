Project Overview

What is a Quantum Network?

A quantum network connects quantum nodes (stationary qubits) through quantum channels to distribute entanglement and quantum information. Unlike classical networks, quantum networks can leverage unique quantum properties like:

  • Quantum entanglement for secure communication
  • Quantum teleportation for state transfer
  • Distributed quantum computing capabilities

Intuitive GUI

Visual design of quantum networks with customizable node properties

Protocol Simulation

Implementation of various entanglement generation protocols

Result Analysis

Visualization of simulation metrics like fidelity and rate

Project Inspiration

This project is based on the framework described in "Remote-Entanglement Protocols for Stationary Qubits with Photonic Interfaces" paper, which provides a modular approach to quantum network protocols. Our simulator brings this theoretical framework to life with an interactive visual interface.

How Can Quantum Network Simulations Help?

Network Architecture and Design

Quantum network simulators help researchers explore topologies and architectures that allow:

  • Scalability across different distances and node counts
  • Optimization of entanglement distribution paths
  • Efficient resource allocation in complex networks
  • Long-distance communication through quantum repeaters

Experiment Planning and Validation

Before building expensive quantum hardware, simulators allow researchers to:

  • Facilitate system integration with hundreds of optical components
  • Easily tune parameters to optimize performance
  • Predict and validate experimental results
  • Identify potential issues before physical implementation

Protocol and Control Plane Design

Simulators are essential for developing the "software" of quantum networks:

  • Evaluate performance of specialized network protocols
  • Test entanglement purification methods
  • Develop quantum routing algorithms
  • Design control systems for managing quantum resources

What to Look for in a Quantum Network Simulator

Accuracy in Modeling

  • The quantum physics - Proper modeling of quantum states, operations, and measurements
  • Sources of uncertainty, error, and noise - Realistic representation of decoherence, loss, and imperfections
  • Precision timing - Accurate timing models for operations and communication

Flexibility and Modularity

A good quantum network simulator should offer flexibility in how one constructs in-simulo components, nodes, networks, and protocols, allowing efficient iteration of designs and configurations.

Scalability

As much as possible, simulators should be scalable to large networks, allowing researchers to study complex network topologies and behaviors.

Levels of Abstraction

Flexibility in the level of abstraction in which a simulation runs:

  • Quantum physics and optics, detailed hardware, elementary entanglement generation
  • Repeater models and protocols - swapping, distillation, etc; small networks
  • Large scale network protocols and behavior

Specifying Parameters of a Simulation

  1. Network topology specification
  2. Properties of optical components
  3. Choose quantum network protocols from a library or define new behavior

Despite the availability of these simulators, there remains a significant gap:

Complete Framework Integration

Our goal is to fully implement the four-layer framework from the "Remote-Entanglement Protocols" paper:

  1. LBB Layer - Logical Building Blocks for protocol topology
  2. Encoding Layer - Qubit representations (Fock, time-bin, polarization)
  3. PBB Layer - Physical Building Blocks for hardware-specific operations
  4. QOM Layer - Quantum Optical Modeling for detailed physics

This will allow users to intuitively design networks at different levels of abstraction, from basic topology to detailed physical parameters.

How the Simulation Works

Quantum Protocol Foundations

Our simulator implements three main protocol topologies as described in the QuREBB framework:

Protocol A: Detection-in-Midpoint

In this topology, both nodes create spin-photon entanglement and send photons to a central detector:

|Ψ⟩A = 1/√2(|0⟩A|0⟩γA + |1⟩A|1⟩γA)

|Ψ⟩B = 1/√2(|0⟩B|0⟩γB + |1⟩B|1⟩γB)

After Bell-state measurement at the midpoint:

|Ψ⟩final = 1/√2(|0⟩A|0⟩B + |1⟩A|1⟩B)

Protocol B: Sender-Receiver

One node creates spin-photon entanglement and sends it to the second node for interaction:

|Ψ⟩initial = 1/√2(|0⟩A + |1⟩A) ⊗ |0⟩B

|Ψ⟩SPE = 1/√2(|0⟩A|0⟩γ + |1⟩A|1⟩γ) ⊗ |0⟩B

|Ψ⟩after-gate = 1/√2(|0⟩A|0⟩γ|0⟩B + |1⟩A|1⟩γ|1⟩B)

Protocol C: Source-in-Midpoint

An entangled photon source in the middle sends photons to both nodes:

|Φ+⟩γ = 1/√2(|0⟩γA|0⟩γB + |1⟩γA|1⟩γB)

|Ψ⟩final = 1/√2(|0⟩A|0⟩B + |1⟩A|1⟩B)

Performance Metrics Calculation

Fidelity Calculation

F = ⟨Ψtarget|ρ|Ψtarget⟩

F = F0 · e-t/T2 · ηtransmission · ηdetection

Where:

  • F0 is the initial fidelity
  • t is the operation time
  • T2 is the coherence time
  • η values are efficiency factors

Rate Calculation

R = Rattempt · Psuccess

Psuccess = ηA · ηB · ηBSM · e-L·α/10

Where:

  • Rattempt is repetition rate
  • ηA, ηB are node efficiencies
  • ηBSM is BSM efficiency
  • L is distance, α is loss coefficient

Quantum Channel Representation

Operations are modeled as quantum channels in density matrix formalism:

ρout = Σi EiρinEi†

For spontaneous emission with loss and decoherence:

ρout = ĈρinĈ†

Ĉ = √pcohĈcoh + √plossĈloss + √pincohĈincoh

The key parameters depend on the physical implementation:

pcoh = ⟨1|ρin|1⟩ · κr/κ · C/(C+1) · Fpγr/(Fpγr + γ*)

Where C is the cooperativity (C = 4g²/κγ) and γ* is the dephasing rate.

From GUI to Simulation

def extract_network_state(self):
    """
    Extract the state of the network as a dictionary of nodes and links.
    """
    nodes = {}
    links = []
    for item in self.items():
        if isinstance(item, NodeItem):
            nodes[item.label_text] = item.get_properties()
        elif isinstance(item, EdgeItem):
            links.append(item.get_properties())
    return {"nodes": nodes, "links": links}

This structured representation contains all quantum properties needed for simulation, which is then passed to the QuREBB framework for accurate physics simulation.
